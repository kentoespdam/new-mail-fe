## Description

Fitur unggah lampiran yang divalidasi langsung berdasarkan *File Rules* dari Master Data Admin.

## Technical Details

- **File Rules Endpoint**: `GET /api/v1/file-rules`
- **Upload Method**: `POST /api/v1/mails/{id}/attachments` (multipart/form-data)
- **Validation**: Client-side Zod schema checking file extension and max size
- **UI**: Progress bar for large file uploads

## Implementation Notes

### File Rules Integration

```typescript
// Fetch file rules
const { data: fileRules } = useQuery({
  queryKey: ['fileRules'],
  queryFn: () => fileRulesService.getActiveRules(),
});

// Zod validation schema
const fileUploadSchema = (rules: FileRules) => 
  z.object({
    file: z
      .instanceof(File)
      .refine((f) => rules.allowedExtensions.includes(getExtension(f.name)), {
        message: `File type not allowed. Allowed: ${rules.allowedExtensions.join(', ')}`,
      })
      .refine((f) => f.size <= rules.maxSizeBytes, {
        message: `File too large. Maximum size: ${formatBytes(rules.maxSizeBytes)}`,
      }),
  });
```

### Upload with Progress

```typescript
const uploadMutation = useMutation({
  mutationFn: async ({ mailId, file }: { mailId: string; file: File }) => {
    const formData = new FormData();
    formData.append('file', file);
    return mailService.uploadAttachment(mailId, formData, {
      onUploadProgress: (progressEvent) => {
        // Update progress UI
      },
    });
  },
});
```

### File Rules Response Interface

```typescript
interface FileRules {
  id: string;
  allowedExtensions: string[]; // ['.pdf', '.doc', '.docx', '.jpg', '.png']
  maxSizeBytes: number;
  isActive: boolean;
  description?: string;
}
```
