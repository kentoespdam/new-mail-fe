# Issue #3.2: Reusable Delete Confirmation Dialog

**Epic:** EPIC 3 - Admin Foundation (Master Data)  
**Parent Issue:** [Issue #3](../0-blueprint/3-crud-master-data.md)

## Description

Build a reusable delete confirmation dialog for all CRUD modules. Adopts GitHub's confirmation pattern: users **must type `HAPUS`** manually before the confirm button activates, preventing accidental clicks.

## Tasks

- [ ] Create `<DeleteConfirmDialog>` component based on ShadcnUI `AlertDialog`
- [ ] Component props:
  - `isOpen: boolean` — visibility control
  - `onClose: () => void` — callback on cancel/close
  - `onConfirm: () => Promise<void>` — callback on confirm (triggers DELETE mutation)
  - `itemName: string` — entity name to display in message
- [ ] Provide `<Input>` in dialog body with placeholder: `Ketik "HAPUS" untuk mengonfirmasi`
- [ ] **"Confirm Delete"** button is **disabled** while input value !== `"HAPUS"`
- [ ] Apply `isPending` state on confirm button to show spinner/loading during DELETE process
- [ ] Styling follows 60:30:10 rule: confirm button uses `bg-red-600` (destructive), cancel button uses `variant="outline"`

## Acceptance Criteria

- Dialog cannot be confirmed without typing `HAPUS` exactly
- Confirm button shows visual feedback during deletion process
- Component is reusable across all CRUD pages (Mail Type, Category, File Rules, etc.) without code duplication

## Design Notes

- Use `bg-red-600` for destructive action button (deviation from accent for safety)
- Cancel button uses neutral outline variant
