# Issue #3.4: Mail Type Management (CRUD)

**Epic:** EPIC 3 - Admin Foundation (Master Data)  
**Parent Issue:** [Issue #3](../0-blueprint/3-crud-master-data.md)

## Description

Implement Mail Type management (e.g., Internal, External) as the primary categorization for correspondence.

## Dependencies

- [Issue #3.1: Admin Data Table & CRUD Layout](./3.1-admin-data-table.md)
- [Issue #3.2: Delete Confirmation Dialog](./3.2-delete-dialog.md)
- [Issue #3.3: Form Input Controls](./3.3-form-inputs.md)

## Tasks

- [ ] Define Zod schema `mailTypeSchema` (Name max 32 characters)
- [ ] Connect TanStack Query with endpoints:
  - `GET /api/v1/mail-types` (List Types)
  - `POST /api/v1/mail-types` (Create Type)
  - `PUT /api/v1/mail-types/{id}` (Update Type)
  - `DELETE /api/v1/mail-types/{id}` (Delete Type)
- [ ] Create ShadcnUI modal form for Create/Edit actions using `<ControlledInput>` from Issue #3.3
- [ ] Use `<DeleteConfirmDialog>` from Issue #3.2 for delete actions

## Acceptance Criteria

- Admin can add, edit, and delete Mail Types
- Data automatically refreshes after successful mutations (Query Invalidation)
- Delete confirmation requires typing `HAPUS`

## Design Notes

- Follow 60:30:10 color rule
- Use `bg-slate-50` (60%) for page background
