# Issue #3.3: Reusable Form Input Controls (Shadcn + RHF + Zod)

**Epic:** EPIC 3 - Admin Foundation (Master Data)  
**Parent Issue:** [Issue #3](../0-blueprint/3-crud-master-data.md)

## Description

Build a collection of reusable input components for all CRUD forms, following the official **shadcn/ui React Hook Form** integration pattern. Each input wraps ShadcnUI with `<Controller>` from React Hook Form to automatically connect to Zod schemas and display error messages.

## Tasks

- [ ] Create `src/components/ui/controlled-form.tsx` with the following controlled components:
  - `<ControlledInput>` — wraps `<Input>` with `<Controller>`, `<FieldLabel>`, and `<FieldError>`
  - `<ControlledTextarea>` — wraps `<Textarea>` with same pattern
  - `<ControlledSelect>` — wraps `<Select>` (using `field.value` & `field.onChange`, not spread props)
  - `<ControlledSwitch>` — wraps `<Switch>` for boolean toggles (e.g., `IsActive`)
- [ ] Each component accepts:
  - `name: string` — field path in form schema
  - `control: Control<TFieldValues>` — from `useForm().control` (use proper typing, not `any`)
  - `label: string` — label text
  - Additional props passed through to underlying ShadcnUI component (using `Omit<ComponentProps, ...>`)
- [ ] Use `Controller` + `Field` + `FieldError` pattern per shadcn/ui documentation:
  ```tsx
  <Controller
    name={name}
    control={control}
    render={({ field, fieldState }) => (
      <Field data-invalid={fieldState.invalid}>
        <FieldLabel htmlFor={field.name}>{label}</FieldLabel>
        <Input {...field} id={field.name} aria-invalid={fieldState.invalid} />
        {fieldState.invalid && <FieldError errors={[fieldState.error]} />}
      </Field>
    )}
  />
  ```
- [ ] Configure `useForm` with `mode: "onChange"` for real-time Zod validation
- [ ] Provide usage example in one CRUD page (e.g., Mail Type form)

## Acceptance Criteria

- Input components automatically display errors from Zod schema when values are invalid
- Form can be reset via `form.reset()` and all inputs return to default values
- No `<Controller>` code duplication across CRUD forms
