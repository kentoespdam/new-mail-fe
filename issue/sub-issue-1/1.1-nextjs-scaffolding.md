### **Issue #1.1: Next.js 16.2.2 Scaffolding & Feature-Based Architecture**

**Epic**: EPIC 1 - Init & Infrastructure
**Parent Issue:** [Issue #2](../0-blueprint/1-init-infrastructure.md)

**Description:**
Menyiapkan struktur proyek menggunakan Next.js 16.2.2 dengan pola folder *feature-based* untuk memastikan skalabilitas dan kemudahan navigasi kode.

**Tasks:**
- [ ] Inisialisasi proyek Next.js 16.2.2 menggunakan TypeScript.
- [ ] Hapus folder `src/app/api` (karena fungsionalitas proxy akan dipindah ke root `proxy.ts`).
- [ ] Buat struktur folder utama:
  - `src/features/`: Untuk logika bisnis per modul (mail, admin, auth).
  - `src/services/`: Untuk `api-client` yang mengarah ke endpoint `/api/mail-service`.
  - `src/lib/`: Untuk konfigurasi infrastruktur (Appwrite, JWT Cache).
  - `src/validations/`: Untuk skema **Zod** berdasarkan OpenAPI.
- [ ] Konfigurasi `path alias` di `tsconfig.json` (e.g., `@/features/*`, `@/components/*`).

**Acceptance Criteria:**
- Struktur folder sesuai dengan rencana *Clean Architecture*.
- Project dapat di-*build* tanpa error.
