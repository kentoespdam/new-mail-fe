### **Issue #1.4: Appwrite REST Client & JWT Cache Utility**

**Epic**: EPIC 1 - Init & Infrastructure
**Parent Issue:** [Issue #2](../0-blueprint/1-init-infrastructure.md)

**Description:**
Membangun *wrapper* untuk Appwrite REST API v1.3.x dan sistem *caching* JWT untuk menghindari blokir *rate limit*.

**Tasks:**
- [ ] Buat `src/lib/appwrite-rest.ts` untuk mengelola request ke endpoint `/account` Appwrite.
- [ ] Implementasikan fungsi `getValidJWT()` di `src/lib/jwt-cache.ts`:
  - Cek apakah JWT masih tersedia di memori server.
  - Jika expired (atau sisa < 1 menit dari limit 15 menit Appwrite), panggil `POST /account/jwt`.
  - Simpan JWT baru ke cache.
- [ ] Pastikan header `X-Appwrite-Project` terkonfigurasi dengan benar.

**Acceptance Criteria:**
- Sistem dapat menghasilkan JWT secara dinamis.
- Panggilan JWT tidak terjadi berulang kali pada setiap request (ter-cache).
