# Issue #2.5: Global Error & Timeout Handling in Proxy

**Epic:** EPIC 2 - Security & Authentication Gate  
**Parent Issue:** [Issue #2](../0-blueprint/2-authentication-authorization.md)

## Description

Handle connection failures to the external API or Appwrite session issues to prevent application crashes.

## Tasks

- [ ] Add global `try-catch` block in `proxy.ts`
- [ ] Handle `401 Unauthorized` error from Appwrite by clearing session cookie and redirecting user to login page
- [ ] Implement timeout on requests to external server (`192.168.1.214`) to prevent hanging if server is down
- [ ] Send user-friendly error message to frontend when system failures occur

## Acceptance Criteria

- Application remains stable even when external backend is unreachable
- Users receive clear information when their session expires
- Requests timeout gracefully instead of hanging indefinitely
