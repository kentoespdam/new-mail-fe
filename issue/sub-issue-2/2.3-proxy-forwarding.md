# Issue #2.3: Root `proxy.ts` - Request Forwarding Engine

**Epic:** EPIC 2 - Security & Authentication Gate  
**Parent Issue:** [Issue #2](../0-blueprint/2-authentication-authorization.md)

## Description

Set up `proxy.ts` at the root level as the central controller for forwarding requests to the PERUMDAMTS external API.

## Tasks

- [ ] Set up main handler in `proxy.ts` to intercept path `/api/mail-service/*`
- [ ] Extract `appwrite-session` from cookie to verify user identity
- [ ] Obtain valid JWT using `getValidJWT` function
- [ ] Forward request using `fetch` to `http://192.168.1.214:8081/api/v1/[path]`
- [ ] Inject header `Authorization: Bearer <JWT>` into forwarded request
- [ ] Handle forwarding for various content types, especially `application/json` and `multipart/form-data` for file uploads

## Acceptance Criteria

- Frontend can access data from external server via `/api/mail-service/` route
- Bearer token is automatically and securely injected server-side
- All content types (JSON, multipart form data) are properly forwarded

## Notes

- Never use direct backend IP in frontend code
- All backend communication must flow exclusively through `/api/mail-service/`
