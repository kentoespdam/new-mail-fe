# Issue #2.4: Proxy-Level Authorization (RBAC with Labels)

**Epic:** EPIC 2 - Security & Authentication Gate  
**Parent Issue:** [Issue #2](../0-blueprint/2-authentication-authorization.md)

## Description

Add authorization layer in `proxy.ts` based on user `labels[]` from Appwrite to protect sensitive endpoints.

## Tasks

- [ ] Create utility function to fetch user profile data (including labels) from Appwrite via REST API
- [ ] Implement access control checks in `proxy.ts`:
  - If path contains `/mail-types`, `/mail-categories`, or `/file-rules`, ensure user has `admin` label
  - If user lacks required label, return `403 Forbidden` response
- [ ] Ensure `staff` role retains access to standard modules like `/mails` and `/mail/folders`

## Acceptance Criteria

- Users without `admin` label are blocked when attempting to access or modify master data
- System provides appropriate error responses (401 for unauthenticated, 403 for unauthorized)
- Staff users retain access to mail and folder modules

## Notes

- RBAC is enforced via Appwrite `labels[]`:
  - `admin` → `/mail-types`, `/mail-categories`, `/file-rules`
  - `staff` → `/mails`, `/mail/folders`
