# Issue #2.2: JWT Lifecycle & Smart Caching Logic

**Epic:** EPIC 2 - Security & Authentication Gate  
**Parent Issue:** [Issue #2](../0-blueprint/2-authentication-authorization.md)

## Description

Build utilities to manage Appwrite JWT generation and server-side caching to avoid rate limit errors (429).

## Tasks

- [ ] Create `getValidJWT(sessionId)` function in `src/lib/jwt-handler.ts`
- [ ] Implement server-side in-memory cache storage (e.g., using `Map` or `lru-cache` library)
- [ ] Implement JWT retrieval logic:
  - If JWT exists in cache and remaining lifetime > 2 minutes, use cached version
  - If expired or missing, call Appwrite REST API `POST /account/jwt`
- [ ] Store new JWT in cache with creation timestamp

## Acceptance Criteria

- JWT is only refreshed when approaching expiration (15 minutes)
- No excessive calls to `/account/jwt` endpoint
- Rate limiting is mitigated via server-side caching

## Notes

- Cache should be server-side only, never exposed to client
- Consider using existing `src/lib/jwt-cache.ts` file as specified in parent issue
