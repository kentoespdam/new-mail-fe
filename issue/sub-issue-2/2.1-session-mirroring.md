# Issue #2.1: Session Mirroring & Server-Side Cookie Management

**Epic:** EPIC 2 - Security & Authentication Gate  
**Parent Issue:** [Issue #2](../0-blueprint/2-authentication-authorization.md)

## Description

Implement authentication system where Appwrite sessions are managed server-side using HttpOnly Cookies following Next.js 16.2.2 security standards.

## Tasks

- [ ] Create `login` Server Action that accepts email and password
- [ ] Use Appwrite REST API `POST /account/sessions/email` to obtain session
- [ ] Extract `expire` property from Appwrite response and set as expiry for `appwrite-session` cookie in Next.js
- [ ] Configure cookie with attributes: `httpOnly: true`, `secure: true`, `sameSite: 'lax'`
- [ ] Create `logout` Server Action to delete session in Appwrite and clear local cookie

## Acceptance Criteria

- User successfully logs in and session cookie appears in Browser DevTools with `httpOnly` flag
- Cookie expiry is synchronized with Appwrite session expiry

## Notes

- Use Appwrite REST API client for full control over request headers
- Follow 60:30:10 color rule if any UI components are created (e.g., login form)
